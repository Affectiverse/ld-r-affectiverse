Architecture

LD-R consists of 6 main component layers:

1 Dataset
This layer handles events dealing with a set of resources. The unit of action here is a named graph.
2 Resource
This layer handles events related to a resource (i.e. anything we would like to describe). Resource URI is used here for identifying the resource.
3 Property
This layer handles events related to properties of a given resource. It can deal with either an individual property or aggregate properties (i.e. combining two or more properties).
4 Object
This layer handles the value(s) of a given property from a given resource. It can be either applied to each individual value of a property or on an aggregate basis to the whole values of a property (e.g. in case of a Map or Chart).
5 View/Edit/Browse Selector
The task of this layer is to select the right component to view, edit or browse object value(s). It will be the hub for connecting new components to our platform.
6 View/Edit/Browse
This layer is responsible for the final presentation of the components. This will be the terminal for user-generated components.
