Config: Properties

Properties config are defined per graph. The 'generic' property config will be used if no specific config is defined for a graph. For properties config, you have to change the following propertiesConfig in configs/reactor.js:

Parameter Description Example value
resourceFocusType Use this, if you want to focus on properties of specific resource types (i.e. RDF Classes). Multiple entity types can be also set. If no value is set, all existing entity types will be shown. ['http://rdfs.org/ns/void#Dataset']
readOnly If set to '1', will disable editing property values. 0
useCategories If set to '1', you can categorize properties in different tabs. It is required to define the list of categories beforehand by using categories. 1
categories List of categories to be used. ['general', 'content', 'structural']

After defining the above general property configs, users can define specific configs for each property by using the URI of the property. Main property configs are as following:

Parameter Description Example value
category If useCategories is set to '1', here you have to mention the category name from the list of categories. ['general']
label Label for the property. If not set, the exact property name for its URI will be used. ['Dataset Title']
hint A hint for the property which will be shown in the popover. ['The title of the dataset described by this document.']
isHidden Hide the property. 1
defaultValue default value for the corresponding component. default
allowNewValue If set to '1', will allow adding new objects to a given property. 1
allowExtension If set to '1', will allow adding new properties to a given object (works only for URI object values). You can specify the list of property extensions and their config in extensions. 1
hasBlankNode If set to '1', will automatically expand blank nodes and will hide the object value of the blank node property on view/edit. 1
objectReactorType Selects the right component for handling object values. Can be set to either 'IndividualObjectReactor' or 'AggregateObjectReactor'. ['IndividualDataEdit']
dataEditType Selects the right component to dispatch edits. ['IndividualObjectReactor']
editor Specifies the right edit component. ['BasicIndividualInput']
editorI Specifies the right edit component for individual objects in case an aggregate object editor is used. ['DBpediaInput']
extendedEditor Specifies the right view when component value is edited in extended mode. ['BasicIndividualDetailEdit']
dataViewType Selects the right component to dispatch views. ['IndividualDataView']
viewer Specifies the right view component. ['BasicIndividualView']
viewerI Specifies the right view component for individual objects in case an aggregate object viewer is used. ['BasicDBpediaView']
extendedViewer Specifies the right view when component value is extended. ['BasicIndividualDetailView']

A sample config is provided at configs/reactor.sample.js. Each component might have its own proprietary configs. You can find the list of proprietary configs for components in Components section.
