Config: General

For general config, you have to change the following parameters in configs/reactor.js:

Parameter Description Example value
appFullTitle The title which appears as page title of your application. ['Linked Data Reactor']
appShortTitle Short title of your application. ['LD-Reactor']
defaultGraphName Name of the default graph to be viewed/browsed/edited. If not set, will consider all existing graph names in 'generic' endpoint. Leave it empty if you want to browse multiple graphs. ['http://dbpedia.org']
resourceFocusType If you want to focus on specific resource types (i.e. RDF Classes) to to be viewed/edited. Multiple entity types can be also set. If no value is set, all existing entity types will be shown. ['http://rdfs.org/ns/void#Dataset', 'http://dbpedia.org/ontology/Organisation']
maxNumberOfResourcesOnPage Maximum number of resources on page. [20]
enableAuthentication If set to '1', will prevent access to datasets unless user is registered and logged in to the system. 0
authGraphName Name of the graph used to store user's data. It is required if enableAuthentication is set to '1'. https://ld-r.org/users
enableUserConfirmation If set to '1', will allow super users to confirm and activate registered users. 0
enableEmailNotifications If set to '1', will enable email notifications on new user registration, user account activation, etc. 0
dynamicResourceDomain Used as domain name when creating random resource URIs. http://example.com
enableLogs If set to '1', will log all update actions of users in the 'log' directory. 0
